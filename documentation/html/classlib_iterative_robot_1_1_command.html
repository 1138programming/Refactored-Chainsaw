Detailed Description

The Command class is the base class for all commands. Commands implement a small bit of functionality into one or more subsystem, and are scheduled by the EventScheduler. Commands are added to the EventScheduler when their run() method is called. Once they are added to the EventScheduler, their canRun() method is called and the command starts if it returns true. Once a command starts, its initilize() function is called once and then their execute() method is called repeatedly. After each time the execute() method is called, the command's isFinished() method is called and the command ends when it returns true. After a command has finished, its end() function is called.
