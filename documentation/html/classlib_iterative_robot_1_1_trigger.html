Detailed Description

The Trigger class is used for running or stopping Commands and CommandGroups based on the state of the Trigger.

The Trigger can be in one of two states: active, or inactive. It has four events associated with it. These events are when the trigger is deactivated, or the transition from active to inactive; when the trigger is activated, or the transition from inactive to active; when the trigger is active; and when the trigger is inactive. Each of these events can run or stop Commands and CommandGroups. At least one of the events associated with the Trigger will constantly be called, usually either the Trigger being inactive or active.
